# Замена байтовых последовательностей в файле (Summer Practice Task 1)

## 📌 Общее описание
Данный проект представляет собой утилиту командной строки на языке C, предназначенную для поиска и замены произвольных байтовых последовательностей в файлах.

Ключевой особенностью реализации является **потоковое чтение файла блоками** заданного размера (по умолчанию 4096 байт). Это позволяет эффективно и с минимальным потреблением оперативной памяти обрабатывать файлы, размер которых значительно превышает объем доступной RAM (например, гигабайтные дампы памяти или медиафайлы).

Алгоритм корректно обрабатывает случаи, когда искомая последовательность (паттерн) находится на границе двух читаемых блоков, гарантируя отсутствие пропусков или некорректных замен.

## 🛠️ Структура проекта
- `app/` — исходный код основного приложения (парсинг аргументов командной строки, валидация ввода, запуск процесса).
- `lib/` — исходный код библиотеки, реализующей низкоуровневый алгоритм поиска и замены (`search_and_replace`).
- `cmake/` — вспомогательные скрипты CMake для упрощения и автоматизации сборки.
- `CMakeLists.txt` — корневой файл конфигурации сборки, связывающий все части проекта воедино.

## ⚙️ Требования
- Компилятор C (GCC, Clang или MinGW для Windows).
- Система сборки CMake (версии 3.10 или выше).

## 🚀 Сборка и запуск

1. Создайте директорию для сборки и перейдите в нее:
```bash
mkdir build
cd build
```

2. Сконфигурируйте проект с помощью CMake. 
   Вы можете задать свой размер блока чтения через параметр `-DBLOCK_SIZE` (в байтах). Если параметр не указан, используется безопасное значение по умолчанию (4096).
```bash
# Для Linux / macOS
cmake -DBLOCK_SIZE=1024 ..

# Для Windows (MinGW)
cmake -G "MinGW Makefiles" -DBLOCK_SIZE=1024 ..
```

3. Запустите процесс компиляции:
```bash
cmake --build .
```

4. Запуск программы:
```bash
# Формат: task1 <входной_файл> <выходной_файл> <hex_паттерн> <hex_замена>
./task1 input.bin output.bin "deadbeef" "00112233"
```

> **Обратите внимание:** паттерн и замена передаются в виде HEX-строк (без префикса `0x`), где каждая пара символов соответствует одному байту.

## 🧠 Как это работает (алгоритм)
1. Программа открывает файлы в бинарном режиме и читает входной файл блоками по `N` байт.
2. Для корректной обработки паттернов, пересекающих границу блоков, используется буфер, позволяющий хранить "хвост" предыдущего чтения.
3. После чтения нового блока в буфере выполняется поиск паттерна с помощью алгоритма посимвольного сравнения.
4. Найденные совпадения заменяются, а "безопасная" часть буфера (которая уже гарантированно не содержит часть паттерна на стыке блоков) записывается в выходной файл.
5. Процесс повторяется до достижения конца файла (EOF), после чего файлы корректно закрываются, а вся выделенная динамическая память освобождается.
